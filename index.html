1. Express模块（框架）

把原生的方法进行封装 日常开发中比较通用的方法都进行了封装，便于开发网站

安装命令
npm install express –save

2. Express-generator 模块

快速生成网站的工具,能够自动生成express模板

安装命令

npm install express-generator -g

查看版本

express –version

生成express模板

express webApp

生成的目录详情

3. res 响应对象

3.1 send()

可以向客户端发送任意类型的数据，一个router中只能有一个send（）方法

3.2 download()

下载文件

3.3 redirect()

重定向 跳转到其他页面，重定向到从指定的URL路径(浏览器地址变为设置的地址)

3.4 render()

视图模板，将视图和数据合并后发送给客户端。常用的模板引擎jade,ejs等,这里使用ejs模板引擎

下载ejs模块
npm install ejs –save

3.5 其他api

响应对象完整api

4 req请求对象

req对象包含了一次请求中的所有数据（http头信息、请求参数…）

4.1 query

获取地址中的参数，即url中?后面的参数

4.2 params

获取地址栏中的:后面的参数,如/user/:id

4.3 body

获取请求体中的数据，一般表单使用post提交时获取表单的数据，需要使用body-parser模块来
解析请求体，才能获取数据，不然是 undefined

4.4 其他api

请求对象完整api

5. 中间件

中间件是一个函数，可以看做过滤器。使用use(fn(req,res,next))来使用中间件。如果当前中间件没有终结请求-响应循环，则必须调用 next() 方法将控制权交给下一个中间件，否则请求就会挂起。

5.1 第三方中间件

通过使用第三方中间件从而为 Express 应用增加更多功能。
(中间件列表)[https://github.com/senchalabs/connect#middleware]

Express路由 Router

路由是指如何定义应用的端点（URIs）以及如何响应客户端的请求
express.Router 类可以创建模块化（独立的）、可挂载的路由对象。Router 对象是一个完整的中间件和路由系统，因此常称其为一个 “mini-app”。

1. console.time()和console.timeEnd()

作用：计算一段代码执行的时间

2. NodeJs作用域和全局对象

在NodeJs中定义的变量，不使用var时，该变量时全局的
NodeJs中的全局对象(函数)是global

3. events事件模块

作用：用来定义事件的基本信息和行为

3.1 绑定事件监听的方法on()、addListener()、once()

on(event,listener)等价addListener(event,listener)
指定事件注册一个监听器，接受一个字符串 event 和一个回调函数

once(event,listener)为指定事件注册一个单次监听器，即 监听器最多只会触发一次，触发后立刻解除该监听器

3.2 删除事件监听的法removeListener()、removeAllListeners()

removeListener(“eventname”,fnname);移除指定事件的某个监听器，监听器 必须是该事件已经注册过的监听器

removeAllListeners(“eventname”);移除所有事件的所有监听器， 如果指定事件，则移除指定事件的所有监听器

3.3 设置事件的最大监听函数setMaxlisteners(n)

默认情况下， EventEmitters 如果你添加的监听器超过 10 个就会输出警告信息。 setMaxListeners 函数用于提高监听器的默认限制的数量

3.4 获取指定事件的监听数组listeners(“eventname”)

返回指定事件的监听器数组

4. http模块

作用：创建服务器、客户端。提供http服务

5. fs文件模块

作用：可以对文件进行操作

6. 同步和异步

同步执行：代码从上往下执行，如果前面一个没有执行完成 后面的都要停下来等待 阻塞
异步执行：并行执行 不需要等待 非阻塞
