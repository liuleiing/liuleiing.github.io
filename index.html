1. background

background是复合属性，包括了background-(color|image|repeat|attachment|position)

书写与顺序无关，可以多个用空格隔开，也可以写一个。复合属性会覆盖单个属性，所以单个属性一般放在复合属性的下面

1.1 background-color

语法:background-color:transparent|color

transparent:表示背景色透明

1.2 background-image

语法:background:none|url()

1.3 background-repeat

语法： background-repeat :repeat| no-repeat| repeat-x| repeat-y

repeat-x:表示背景图横向平铺

repeat-y : 背景图像在纵向平铺

需要先设置背景图片(background-image)之后设置才是否平铺

1.4 background-position

语法:position : top | center | bottom | left | center | right

如果只指定了一个值，该值将用于横坐标。纵坐标将默认为50%。第二个值是纵坐标。 值可以写英文，也可以写数字，甚至可以是负数。常用于 CSS sprites（CSS精灵）

该属性定位不受对象的补丁属性(padding)设置影响

1.5 background-attachment

语法： background-attachment :scroll| fixed

scroll : 背景图像是随对象内容滚动

fixed : 背景图像固定

2. background-CSS3

css3中新增的背景属性

2.1 background-size

设置背景图片的大小

语法： = [ | | auto ]{1,2} | cover | contain

cover:撑满整个容器,背景图像有可能超出容器

contain:将背景图像等比缩放到宽度或高度与容器的宽度或高度相等，背景图像始终被包含在容器内

2.2 background-origin

设置图片的原点位置，即background-position的起始位置

语法:background-origin:padding-box|border-box|content-box

padding-box:默认值。
border-box:原点位置从border开始
content-box:原点位置从content开始

1. Ajax原理

通过浏览器的javascript对象XmlHttpRequest(Ajax引擎)对象来向服务器发异步请求并接收服务器的响应数据，然后用javascript来操作DOM而更新页面

2. Ajax引擎

所有现代浏览器Ajax引擎指的是XMLHttpRequest对象，是一个js对象，老版本的是ActiveObject对象。

真正的请求是由Ajax引擎发出，不是浏览器窗口发出，所以浏览器窗口是不会刷新的。

Ajax引擎同时也接收服务器响应的数据。

3. 使用Ajax引擎（XMLHttpRequest对象）

常用语法

3.1 Ajax使用get方式发送请求
3.2 Ajax使用post方式发送请求

4. onreadystatechange事件属性

当Ajax引擎的状态发生改变时都会执行onreadystatechange属性对应的方法

状态属性readyState:

  • 0:请求未初始化
  • 1:服务器连接已建立
  • 2:请求已接收
  • 3:请求处理中
  • 4:请求已完成，且响应已就绪

5. Ajax实现同步和异步

同步是阻塞模式，异步是非阻塞模式

ajax引擎读对象.open(方式get/post，url地址，同步异步). 第三个参数为true表示异步,默认为true，第三个参数为false表示同步

6. eval()

将字符串当成js代码来执行，可用来将json字符串解析成json对象

2. Mongodb安装

MongoDB是一个基于分布式文件存储的数据库，由c++语言编写。文件存储格式为BSON(json的扩展)
BSON是一种类似json的二进制形式的存储格式，简称Binary JSON，它和JSON一样，支持内嵌的文档对象和数组对象，但是BSON有JSON没有的一些数据类型，如Date和BinData类型

MongoDB官网地址

MongoDB下载地址

由于我的操作系统是win32,所以以下配置步骤适用于win32的系统

  1. 以管理员身份运行cmd，
  2. 要想执行mongodb的命令，需要cd到他的安装目录下面的bin文件夹下
  3. 在想建立数据库盘上(比如：d盘)创建文件夹mongodb,并且建立子目录db和log
  4. 把mongodb文件夹的只读属性去掉

  5. 执行命令：mongod –dbpath “d:\mongodb\db” –logpath “d:\mongodb\log\MongoDB.log” –logappend (win32系统)

    mongod –storageEngine mmapv1 –dbpath “d:\mongodb\db” –logpath “d:\mongodb\log\MongoDB.log” –logappend (win64系统)

  6. 以管理员身份运行cmd，进入bin文件夹，执行命令: mongod –dbpath “d:\mongodb\db” –logpath “c:\mongodb\log\MongoDB.log” –logappend –install –serviceName “MongoDB”
  7. 启动mongodb服务:net start MongoDB

3. Mongodb基本使用

以下命令都在Mongodb的bin目录下的mongo.exe中执行。（因为没有配置环境变量）

作用 命令
显示数据库列表 >show dbs
创建(使用)数据库 >use dbname
显示集合 >show collections
增加数据 >db.collectionsName.save({“key”:”value”})
>db.collectionName.insert({“key”:”value”})
删除文档 >db.collectionsName.remove({})
删除集合 >db.users.drop()或>db.runCommand({“drop”,”users”})

4.Mongodb条件查询
查找key=value的数据 db.collection.find({“key”:value})
key>value db.collection.find({“key”:{$gt:value}})
key<value db.collection.find({“key”:{$lt:value}})
key>=value db.collection.find({“key”:{$gte:value}})
key<=value db.collection.find({“find”:{$lte:value}})
value1<key<value2 db.collection.find({“key”:{$lt:value2,$gt:value1}})
key<>value db.collection.find({“key”:{$ne:value}})
取模运算 key%10==1 db.collection.find({“key”:{$mod:[10,1]}})
key等于[1,2,3]中任何一个 db.collection.find({“key”:{$in:[1,2,3]}})
key不等于[1,2,3]中任何一个 db.collection.find({“key”:{$nin:[1,2,3]}})
key对应的值的数量是1（值必须是数组） db.collection.find({“key”:{$size:1}})
true返回存在字段key的数据，false返回不存在字段key的数据 db.collection.find({“key”:{$exists:true\false}})
符合两个条件中任意一个的数据 db.collection.find({$or:[{“key1”:value1},{“key2”:value2}])
内嵌对象中的值匹配 db.collection.find({ “key.subkey” :value })

mongdb其他操作
排序，1代表升序，-1代表降序 db.collection.find().sort({“key1”:-1,”key2”:1})
添加索引,1代表升序，-1代表降序 db.collection.ensureIndex({“key”:1})
控制返回结果数量，如果参数是0，则当作没有约束 db.collection.find().limit(5)
控制返回结果跳过多少数量 db.collection.find().skip(5)
返回结果集的条数 db.collection.find().count(true\false)
格式化 db.conllections.find().pretty()

2. 重复的html的复写

问题描述：

html文件中的有许多的代码都是重复的，如果要对其进行操作，就会对每个html中的相同部分做相同的操作；
文件一多的话，就会浪费很多的时间，而且不利于优化和维护。

解决方法：

将重复的html提出来，放在一个公共的文件中。使用<script src="文件名"></script>导入到页面中响应的位置。

具体步骤：
  1. 将公共的html提出来，放到新建的文件中（比如 navslider.html）
  2. 将 navslider.html里面的内容使用\转义\n,并加上'',变成一个多行字符串，赋值给一个变量txt
  3. 使用document.write(txt);输出到浏览器
  4. 在提出来的代码原来的位置使用<script src="文件名"></script>导入文件
  5. 注意也可以使用正则表达式来生成多行字符串
