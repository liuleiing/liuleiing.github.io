NodeJs学习笔记 05

Share

内容总结

  • Express 模块（框架）
  • Express-generator 模块
  • res 响应对象
  • req 请求对象
  • 中间件

1. Express模块（框架）

把原生的方法进行封装 日常开发中比较通用的方法都进行了封装，便于开发网站

安装命令
npm install express –save

2. Express-generator 模块

快速生成网站的工具,能够自动生成express模板

安装命令

npm install express-generator -g

查看版本

express –version

生成express模板

express webApp

生成的目录详情

3. res 响应对象

3.1 send()

可以向客户端发送任意类型的数据，一个router中只能有一个send（）方法

3.2 download()

下载文件

3.3 redirect()

重定向 跳转到其他页面，重定向到从指定的URL路径(浏览器地址变为设置的地址)

3.4 render()

视图模板，将视图和数据合并后发送给客户端。常用的模板引擎jade,ejs等,这里使用ejs模板引擎

下载ejs模块
npm install ejs –save

3.5 其他api

响应对象完整api

4 req请求对象

req对象包含了一次请求中的所有数据（http头信息、请求参数…）

4.1 query

获取地址中的参数，即url中?后面的参数

4.2 params

获取地址栏中的:后面的参数,如/user/:id

4.3 body

获取请求体中的数据，一般表单使用post提交时获取表单的数据，需要使用body-parser模块来
解析请求体，才能获取数据，不然是 undefined

4.4 其他api

请求对象完整api

5. 中间件

中间件是一个函数，可以看做过滤器。使用use(fn(req,res,next))来使用中间件。如果当前中间件没有终结请求-响应循环，则必须调用 next() 方法将控制权交给下一个中间件，否则请求就会挂起。

5.1 第三方中间件

通过使用第三方中间件从而为 Express 应用增加更多功能。
(中间件列表)[https://github.com/senchalabs/connect#middleware]

6. Express路由 Router

路由是指如何定义应用的端点（URIs）以及如何响应客户端的请求
express.Router 类可以创建模块化（独立的）、可挂载的路由对象。Router 对象是一个完整的中间件和路由系统，因此常称其为一个 “mini-app”。

1. Buffer缓冲区

专门用来存数据的缓冲区

1.1 创建Buffer对象

通过直接new Buffer(num || array || string),定义了长度后不能改变。num表示缓冲区长度，
array表示直接给缓冲区赋数组，string直接给缓冲区赋字符串。

1.2 Buffer写入

通过下标直接赋值

使用write()方法

1.3 读取内容（转换为为字符串）

1.4 拷贝

将scbuff的内容拷贝到tchuff，从schuff的小标1开始到下标3（不包括3）

从tcbuff的下标2开始赋值

2. fs 文件模块加强

fs模块包需要下载到node_module中，通过require("fs")导入才能使用
fs模块大部分的方法都分为同步和异步的方式。异步与同步的方式对比多了一个callback

2.1 读取文件
2.2 写入文件

写入文件是，如果路径文件不存在，会新建一个文件，写入的数据可以是String(字符串)
和Buffer(流)对象

2.3 读取文件信息

stat()读取文件信息，主要用来判断是否是文件

2.4 删除文件

使用unlink()和unlinkSync()来删除某个文件

2.5 获取目录下所有的文件和文件夹

使用readdir()和readdirSync()来获取

2.6 删除空文件夹

使用rmdir()和rmdirSync()来删除空的文件夹

3. 流的概念

一组有序、有起点和终点 字节数据传输方式

3.1 可读流

createReadStream()创建一个可读流，会触发data、end、error等事件

3.2 可写流

createWriteStream()创建可写流，会触发finish、error等事件

3.3 管道方式pipe()

管道(pipe)提供了一个输出流到输入流的机制。通常我们用于从一个流中获取数据并将数据传递到另外一个流中

3.4 链式流

链式是通过连接输出流到另外一个流并创建多个对个流操作链的机制。链式流一般用于管道操作

4. path路径模块

使用var path=require("path")来导入path模块

normalize()

规范化路径,window下是\,Unix下是/。尾部的/不会被去掉

join()

组合参数(必须是字符串的形式)中的路径，返回规范化路径

resolve()

返回一个绝对路径，相当与在cmd中cd每一个参数后获取的绝对路径
如果某个参数是以”/“开头，则返回到根目录继续cd。
遇到"../"返回上级目录再继续cd，遇到"./"不做操作继续cd
末尾的"/"会被干掉

isAbsolute(path)

判断是否是绝对路径

relative(path1,path2)

返回path2相对与path1的路径
即相同的部分用一个”../“代替，path1中多有的不同部分，用相等数量的”../“代替
path2中独有的部分直接写到尾部

dirname(path)

返回文件夹的路径，即返回上级的路径

basename(path,ext)

返回路径的最后一部分，去除末尾和ext相同的部分

extname(path)

返回路径（不是以”.”开头的）中最后一个”.”之后的部分
路径中没有”.”或者是以”.”开头的，返回空字符串

parse(pathstr)

将path字符串，变为path对象

format(pathObj)

将path对象，变为path字符串

1. console.time()和console.timeEnd()

作用：计算一段代码执行的时间

2. NodeJs作用域和全局对象

在NodeJs中定义的变量，不使用var时，该变量时全局的
NodeJs中的全局对象(函数)是global

3. events事件模块

作用：用来定义事件的基本信息和行为

3.1 绑定事件监听的方法on()、addListener()、once()

on(event,listener)等价addListener(event,listener)
指定事件注册一个监听器，接受一个字符串 event 和一个回调函数

once(event,listener)为指定事件注册一个单次监听器，即 监听器最多只会触发一次，触发后立刻解除该监听器

3.2 删除事件监听的法removeListener()、removeAllListeners()

removeListener(“eventname”,fnname);移除指定事件的某个监听器，监听器 必须是该事件已经注册过的监听器

removeAllListeners(“eventname”);移除所有事件的所有监听器， 如果指定事件，则移除指定事件的所有监听器

3.3 设置事件的最大监听函数setMaxlisteners(n)

默认情况下， EventEmitters 如果你添加的监听器超过 10 个就会输出警告信息。 setMaxListeners 函数用于提高监听器的默认限制的数量

3.4 获取指定事件的监听数组listeners(“eventname”)

返回指定事件的监听器数组

4. http模块

作用：创建服务器、客户端。提供http服务

5. fs文件模块

作用：可以对文件进行操作

6. 同步和异步

同步执行：代码从上往下执行，如果前面一个没有执行完成 后面的都要停下来等待 阻塞
异步执行：并行执行 不需要等待 非阻塞
